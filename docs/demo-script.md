## Project Write-up

Paykubo is a Morph-powered marketplace for paid APIs and autonomous agent
payments. For the x402 Agentic Payments track, it solves a common problem in
the Philippines and Southeast Asia: useful API providers, SMEs, and builders
need a simple way to sell digital work without card rails, invoices, or custom
billing integrations. Providers publish existing HTTPS APIs, buyers and agents
receive x402 payment requirements, and successful calls settle in USDC on Morph
Hoodi. Paykubo records receipts, fee splits, provider request traces, managed
credits, agent budgets, and public proof pages so every paid action is
auditable. The result is an API commerce layer where humans, apps, and AI agents
can buy work instantly, one request at a time.

## Pacing Notes

- Do not show provider onboarding, OpenAPI import setup, wallet switching, or
  external service dashboards in the 2-minute submission video.
- Use pre-created listings, orders, receipts, agent runs, and proofs so the
  recording stays focused on product value.
- If a live wallet prompt is slow, use an already completed order and explain
  the payment step while showing the receipt and Morph transaction link.
- Keep the cursor movement deliberate; each screen should communicate one
  judging point: use case, payment flow, receipt, agent payment, or proof.
